Guide Setup

Installation

The goal of installation is simple: get the package into Unity, make sure the editor compiles cleanly, and confirm you can create and execute a graph before you start building real content.

What you need first

  • A Unity project on a recent LTS editor branch.
  • The OctoShaper package or release payload for your current build.
  • The Unity Collections package, which OctoShaper depends on through its package manifest.

Import checklist

  1. Add OctoShaper to the project using the release package, local package checkout, or the distribution method supplied with your build.
  2. Let Unity finish importing and confirm the Console is clear of OctoShaper compile errors.
  3. Create a new graph asset through Assets > Create > OctoShaper > Procedural Graph.
  4. Open the graph asset and confirm the graph editor, sidebar tools, and node search all appear normally.
  5. If you plan to execute graphs through the generated path, run Tools > OctoShaper > Generate Graph Code after editing your graphs.

First validation pass

A clean import is not enough. Make one small graph and confirm the full loop works.

  • Create a graph asset and save it.
  • Add a few nodes and connect them through the main ElementSet flow.
  • Attach a ProceduralGraphExecutor to a scene object and assign the graph.
  • Trigger generation and verify the result appears in the scene or preview workflow you expect.

Common issues

  • If generated execution fails, regenerate graph code so the strict executor registry matches your current graphs.
  • If runtime results are missing, check that the graph main output resolves to an ElementSet.
  • If the editor opens but regeneration never happens, make sure your graph is actually assigned to a ProceduralGraphExecutor or importer workflow.
