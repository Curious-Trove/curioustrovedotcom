Concept Data Model

What Are Elements?

In OctoShaper, procedural data moves through graphs as an ElementSet. Think of it as a row-based dataset: each element is one row, and each attribute is a named column stored across all rows.

An element is not just one thing

Depending on the graph, one element might represent a point in space, a spawn candidate, a placed prefab, a spline sample, or any other unit of procedural work.

What matters is that OctoShaper treats it as a row of related values that nodes can read, filter, extend, or transform.

Attributes are the payload

  • Attributes are stored by name and type.
  • Nodes can create new attributes or overwrite existing ones.
  • ProceduralConfiguration and graph requirements define which values the graph expects to see at the boundary.

How ElementSet flows through a graph

  • The main graph input and main graph output are both ElementSet-based.
  • Generator nodes usually create rows or expand a dataset.
  • Modifier nodes typically add attributes, transform values, or remove rows.
  • Instancing and visualization workflows consume the resulting ElementSet to build Unity-side output.

Why this matters

Once you understand ElementSet, the rest of OctoShaper becomes easier to read. Node signatures make more sense, graph outputs become easier to inspect, and it is clearer why the tool favors attributes and element collections over ad-hoc arrays of unrelated data.

When a graph is difficult to debug, one of the fastest ways to reason about it is to ask: how many elements are flowing here, and which attributes should exist on them right now?
